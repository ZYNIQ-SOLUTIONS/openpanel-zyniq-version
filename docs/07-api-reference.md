## Overview

The API runs on **Fastify 5** and exposes two categories of routes:

1. **Public SDK API** ŌĆö for client SDKs, no authentication required, CORS open
2. **Dashboard API** ŌĆö for the dashboard frontend, requires session cookie, CORS restricted

Base URL: `http://api.yourdomain.com` (or `http://localhost:3333` in dev)

---

## Public SDK Routes

These routes are open to the internet. Any app can call these with a valid Client ID / Secret Key.

### Event Ingestion

#### `POST /event`
Ingest one or more analytics events.

```json
// Request body
{
  "type": "track",
  "payload": {
    "name": "purchase",
    "properties": { "amount": 99, "currency": "USD" }
  }
}
```

#### `POST /track`
Track page views and sessions.

#### `POST /profile`
Create or update a user profile.

```json
{
  "profileId": "user-123",
  "properties": {
    "email": "user@example.com",
    "name": "John Doe"
  }
}
```

### Data Access

#### `GET /export`
Export raw event data. Requires API client authentication.

#### `POST /import`
Import events from CSV or other formats.

#### `GET /insights`
Programmatic access to analytics insights.

#### `GET /manage`
Client/project management operations.

## Dashboard API (tRPC)

The dashboard uses **tRPC v11** over HTTP. All tRPC routes are prefixed with `/trpc/`.

### Authentication
- Cookie-based: `session` cookie (signed, HTTP-only)
- Session validation happens in the `onRequest` hook before any tRPC call

### tRPC Router Structure

Calls follow the pattern: `POST /trpc/<router>.<procedure>`

Key routers include:

| Router | Procedures |
|--------|-----------|
| `organization` | `list`, `create`, `update`, `delete`, `getById` |
| `project` | `list`, `create`, `update`, `delete`, `getById` |
| `client` | `list`, `create`, `update`, `delete` |
| `event` | `list`, `getById`, `deleteMany` |
| `dashboard` | `list`, `create`, `update`, `delete`, `getReports` |
| `user` | `me`, `update`, `invite`, `list` |
| `report` | analytics report queries (funnels, cohorts, etc.) |
| `profile` | user profile lookups |
| `notification` | `list`, `create`, `update` |
| `session` | `list`, `getById` (session replays) |

### Using tRPC from External Clients

You can call tRPC procedures directly via HTTP if needed:

```bash
# Example: list organizations
curl -X POST http://api.localhost:3333/trpc/organization.list \
  -H "Content-Type: application/json" \
  -b "session=<your_session_token>"
```

Or use the tRPC client SDK (`@trpc/client`).

## Live / Real-time Route

#### `WebSocket /live`
WebSocket connection for real-time event streaming to the dashboard.

- Requires session cookie authentication
- Publishes live events as they are ingested
- Dashboard consumes these via `react-use-websocket`

## Authentication Flow

```
1. User opens dashboard (apps/start)
2. Dashboard calls POST /trpc/auth.login with credentials
3. API validates credentials (Prisma ŌåÆ users table, Argon2 password check)
4. API creates session record in PostgreSQL
5. API sets signed `session` cookie (via @fastify/cookie)
6. Subsequent requests include cookie automatically
7. API validates cookie on every request via onRequest hook
8. Session decoded ŌåÆ user context available in tRPC context
```

### Session Token Structure
- Generated by Oslo (`encodeSessionToken`)
- Contains session ID
- Signed with `COOKIE_SECRET`
- Stored in PostgreSQL with expiry

---

## Rate Limiting

Rate limiting is applied via `@fastify/rate-limit`.

Default configuration applies to event ingestion endpoints to prevent abuse.

Per-client rate limits can be configured in the organization/client settings.
